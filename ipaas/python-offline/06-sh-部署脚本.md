# sh - 部署脚本目录详解

## 概述

`sh/` 目录承载部署流程的不同阶段脚本（pre/post/scale），处理配置下发、文件复制、集群扩容等场景。

## 目录结构

```
sh/
├── pre/                          # 部署前准备
│   ├── pre.sh                    # 主脚本
│   ├── all_server_all/           # 复制到所有服务器的组件
│   └── oper_master_all/          # 复制到运维主机的组件
├── post/                         # 部署后检查
│   ├── check.sh                  # 部署验证脚本
│   └── post.sh                   # 后置处理脚本
└── scale/                        # 扩容脚本
    └── scale-component/
        ├── common/sh/            # 实际脚本位置
        │   └── pre.sh            # 扩容准备脚本
        └── all_server_all/       # 待复制组件
```

## 1. pre/ 目录 - 部署前准备

### pre.sh 主功能

#### 1.1 配置替换 (cp_cfg)

用模板替换所有配置文件，将占位符替换为实际环境值。

占位符列表：
- `DB0` → 数据库地址
- `REDIS` → Redis 地址
- `license_host=127.0.0.1` → 许可证服务器 IP
- `name = AUTOEX` → 产品标识名称

#### 1.2 组件复制 (cp_dir)

复制各组件到目标目录：

| 源目录 | 目标位置 | 说明 |
|--------|----------|------|
| `asap-monitor` | `all_server_all/` | 复制到所有服务器 |
| `asap-log-purge` | `all_server_all/` | 复制到所有服务器 |
| `asap_deviceid_report` | `all_server_all/` | 复制到所有服务器 |
| `maxs_status_check` | `oper_master_all/` | 仅复制到运维主机 |
| `asap-process-consume-monitor` | `all_server_all/` | 复制到所有服务器 |

#### 1.3 扩容准备 (prepare_add)

为扩容准备，复制到 `/data/scale/scale/common/`。

#### 1.4 动态配置修改 (change_config)

根据环境动态修改配置：
- 读取数据库/Redis 组信息替换占位符
- 支持 DM8 数据库配置替换（端口 5236，用户 SYSDBA）
- 支持 VastBase 配置（端口 15432）
- 更新 license_host 为实际许可证服务地址

## 2. post/ 目录 - 部署后检查

### check.sh - 部署验证

验证部署结果，使用 Ansible 在 `all_server` 主机组批量执行：

```bash
# 依次执行并检查:
1. log_purge.py 执行
2. deviceid_report.py 执行
3. monitor.py -m a 执行
4. asap_monitor systemd 服务状态
```

全部成功才返回 0，任一失败则返回非 0。

### post.sh - 后置处理

执行部署后的清理和初始化工作。

## 3. scale/ 目录 - 扩容脚本

### scale-component/ 结构

```
scale/
└── scale-component/
    ├── common/sh/
    │   └── pre.sh      # 扩容专用准备脚本
    └── all_server_all/ # 待复制组件
```

### 扩容流程

1. 将 python-offline 复制到 `/data/asap/scale/scale/common/`
2. 将 scale 目录下的脚本覆盖 sh/ 目录
3. 执行扩容准备

## 配置模板说明

所有模板使用占位符，pre.sh 会动态替换为实际环境值：

| 占位符 | 说明 | 示例值 |
|--------|------|--------|
| `DB0` | 数据库地址 | 192.168.1.10 |
| `REDIS` | Redis 地址 | 192.168.1.11 |
| `license_host` | 许可证服务器 | 192.168.1.100 |
| `name` | 产品标识 | AUTOEX |

## 使用场景

### 正常部署流程

```
pre.sh (部署前准备)
    ↓
部署组件到各服务器
    ↓
post.sh + check.sh (部署后检查)
```

### 扩容流程

```
scale-component/pre.sh (为新增服务器准备)
    ↓
将新服务器加入集群
    ↓
check.sh (验证新节点)
```

## 关键设计

1. **模板化配置**: 所有配置文件使用占位符，部署时动态替换
2. **分组部署**: 根据组件类型复制到不同主机组
3. **Ansible 批量执行**: 使用 Ansible 进行批量部署和验证
4. **幂等性**: 支持重复执行，不会重复复制或覆盖
