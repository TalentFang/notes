# python-offline 模块总览

## 概述

`python-offline` 是 ASAP/IPaaS 分布式监控系统的 Python 工具集，提供分布式集群的状态检查、资源监控、日志清理、设备管理等运维功能。

**目标机器**: `root@192.168.11.205`，常用路径 `/data/asap/deploy`

---

## 执行场景汇总

| 场景   | 触发时机    | 命令                                    | 是否systemd |
| ---- | ------- | ------------------------------------- | --------- |
| 部署验证 | 变更后     | `maxs_status_check_tool.py maxs base` | ❌         |
| 故障排查 | 服务异常    | `maxs_status_check_tool.py maxs all`  | ❌         |
| 系统监控 | 开机自启    | `systemctl start asap_monitor`        | ✅         |
| 设备授权 | 开机/定时   | `deviceid_report.py`                  | ❌         |
| 日志清理 | 磁盘满时/定时 | `log_purge.py`                        | ❌         |
| 资源分析 | 定时/手动   | `monitor.py -m a`                     | ❌         |

### 场景 1: maxs_status_check - 状态检查（最常用）

**检查流程**:

### 场景 2: asap-monitor - 系统监控（systemd 服务）

**用途**: 持续运行，开机自启，采集系统指标写入数据库

**路径**: `/data/asap/deploy/python-offline/asap-monitor/`

**systemd 服务**: `asap_monitor.service`

**服务配置** (`/etc/systemd/system/asap_monitor.service`):
```ini
[Unit]
Description=22th:perfstats
RequiresMountsFor=/data

[Service]
EnvironmentFile=/etc/service.env
Type=forking
ExecStart=/bin/sh -c 'source /etc/profile && python3 $DEPLOY_HOME/asap-monitor/perfstats-to-syslog.py&'
ExecStop=/bin/kill -s TERM $MAINPID
Restart=always              # 总是重启
RestartSec=60                # 重启间隔 60 秒
StartLimitBurst=60          # 最多重启 60 次
StartLimitInterval=36000    # 36000 秒内
TimeoutSec=720

[Install]
WantedBy=multi-user.target
```

**关键特性**:
- `Restart=always`: 服务失败后自动重启
- `RestartSec=60`: 重启前等待 60 秒
- `StartLimitBurst=60`: 最多重启 60 次
- `EnvironmentFile=/etc/service.env`: 环境变量文件

**管理命令**:
```bash
# 启动服务
systemctl start asap_monitor

# 停止服务
systemctl stop asap_monitor

# 查看状态
systemctl status asap_monitor

# 查看日志
journalctl -u asap_monitor -f

# 重启服务
systemctl restart asap_monitor

# 查看监控日志
tail -f /data/asap/logs/asap_monitor.log
```

**采集指标**:
- CPU、内存、磁盘、网络、磁盘 IO
- RAID 状态（MegaCLI）
- Flink TaskManager 在线检查

**数据写入**:
- 本地日志: `/data/asap/logs/asap_monitor.log`（滚动，10MB/文件，保留 7 个）
- 数据库表: `SSA.MONITOR_MEM`, `SSA.MONITOR_DISK`, `SSA.MONITOR_IO_DISK`, `SSA.MONITOR_NETWORK`, `SSA.MONITOR_RAID`

### 场景 3: asap-deviceid-report - 设备授权上报

**用途**: 开机时执行，上报设备 ID 到 Redis 用于授权验证

**路径**: `/data/asap/deploy/python-offline/asap-deviceid-report/`

```bash
# 正常运行
python3 deviceid_report.py

# 检查激活状态
python3 deviceid_report.py check
```

**工作流程**:
```
1. 获取设备ID（硬件指纹）
2. 查询数据库许可证
3. 检查设备ID重复（防盗用）
4. 上报 Redis
5. 清理过期数据
```

**配置**: `deviceid_report.cfg`

```ini
[db]
type = vastbase
host = DB0
port = 15432
name = UPMS

[redis]
host = REDIS
port = 6379

[product]
name = AUTOEX
license_host = 127.0.0.1
```

**建议**: 配置 crontab 定时执行或开机触发

### 场景 4: asap-log-purge - 日志清理

**用途**: 磁盘空间不足时自动清理各服务日志

**路径**: `/data/asap/deploy/python-offline/asap-log-purge/`

```bash
# 手动执行
python3 log_purge.py

# 或配置 crontab 定时执行
```

**工作流程**:
```
1. 从数据库读取 DISK_PERCENT 阈值
2. 获取当前 /data 磁盘使用率
3. 若超过阈值，查询日志路径
4. 根据服务类型清理日志
```

**清理策略**:
| 服务 | 方式 |
|------|------|
| spark/flink/livy/hadoop/kafka | 清空目录内所有日志 |
| elasticsearch | 清空主日志 + gc.log* |
| 其他 | 清空单个日志文件 |

**安全特性**: 使用 `cat /dev/null` 而非 `rm`，保持文件权限

### 场景 5: asap-process-consume-monitor - 进程资源监控

**用途**: 监控进程 CPU/内存/IO 使用情况，记录到本地日志

**路径**: `/data/asap/deploy/python-offline/asap-process-consume-monitor/`

```bash
# CPU Top20
python3 monitor.py -n 20 -m cpu

# 内存 Top20
python3 monitor.py -n 20 -m mem

# 全部指标
python3 monitor.py -m a

# IO 监控
python3 io_monitor.py
```

**输出目录**:
- CPU: `/data/asap/asap-process-consume-monitor/logs/cpu/`
- Mem: `/data/asap/asap-process-consume-monitor/logs/mem/`

**日志清理**: 每天凌晨 1 点清理 7 天前日志

## systemd 服务配置说明

### asap_monitor.service 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `RequiresMountsFor` | `/data` | 依赖 /data 挂载 |
| `Type` | `forking` | 进程 fork 后退出 |
| `Restart` | `always` | 始终重启 |
| `RestartSec` | `60` | 重启间隔 60 秒 |
| `StartLimitBurst` | `60` | 10 小时内最多 60 次重启 |
| `TimeoutSec` | `720` | 启动/停止超时 720 秒 |

### 服务管理

```bash
# 开机启用
systemctl enable asap_monitor

# 开机禁用
systemctl disable asap_monitor

# 查看是否启用
systemctl is-enabled asap_monitor
```
