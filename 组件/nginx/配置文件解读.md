下面我将系统性地解释 `nginx.conf` 中涉及到的**核心语法和指令**，按配置结构分类说明：

## 一、全局配置语法

### 1. 基础指令

```nginx
user root;
worker_processes 3;
```

| 指令 | 语法格式 | 作用域 | 说明 |
|:---|:---|:---|:---|
| `user` | `user username [group];` | main | 设置Nginx工作进程的运行用户和组 |
| `worker_processes` | `worker_processes number \| auto;` | main | 工作进程数量，`auto`表示自动匹配CPU核心数 |
| `load_module` | `load_module module_path;` | main | 动态加载第三方模块（如ModSecurity） |

### 2. 注释语法

```nginx
# 这是单行注释
#load_module /path/module.so;  # 被注释的指令不会执行
```

## 二、Events 模块语法

```nginx
events {
    worker_connections 1024;
}
```

| 指令 | 说明 |
|:---|:---|
| `worker_connections` | 每个工作进程的最大并发连接数 |
| `use` | 指定事件模型（如`epoll`，Linux下自动选最优） |
| `multi_accept` | 是否一次接受所有新连接 |

## 三、Stream 模块语法（四层代理）

### 1. 基础结构

```nginx
stream {           # 四层代理上下文
    upstream name { # 上游服务器组
        server ...;
    }
    server {        # 监听配置
        listen port;
        proxy_pass upstream_name;
    }
}
```

### 2. 日志配置

```nginx
log_format upstream_log '$remote_addr [$time_local] ...';
access_log /path/logs/access.log upstream_log;
```

| 指令 | 语法 | 说明 |
|:---|:---|:---|
| `log_format` | `log_format name string;` | 定义日志格式，包含变量和普通字符串 |
| `access_log` | `access_log path [format [buffer=size]];` | 指定访问日志路径和格式 |

### 3. Upstream 语法

```nginx
upstream asap_minio {
    server master-01:9005 max_fails=3 fail_timeout=30s;
    server master-02:9005 weight=2 backup;
}
```

#### Server 指令参数：

| 参数 | 说明 | 示例 |
|:---|:---|:---|
| `weight=number` | 权重，默认1，值越高分配请求越多 | `weight=3` |
| `max_fails=number` | 允许失败次数，超过则标记为不可用 | `max_fails=3` |
| `fail_timeout=time` | 失败超时时间，超时后再次测试 | `fail_timeout=30s` |
| `backup` | 标记为备用服务器，仅当主服务器全不可用时使用 | `backup` |
| `down` | 标记服务器永久不可用 | `down` |

### 4. 负载均衡算法

```nginx
upstream backend {
    # 轮询（默认）- 无参数
    server server1;
    server server2;
    
    # 最少连接数
    least_conn;
    
    # IP哈希（保持会话）
    ip_hash;
    
    # 通用哈希
    hash $remote_addr consistent;  # consistent启用一致性哈希
}
```

### 5. Server 监听语法

```nginx
server {
    listen 9004;           # 监听TCP端口
    listen 192.168.1.1:80; # 指定IP和端口
    proxy_pass asap_minio;  # 转发到upstream
    proxy_timeout 60s;      # 代理超时时间
}
```

## 四、HTTP 模块语法（七层代理）

### 1. HTTP 上下文基础

```nginx
http {
    include       mime.types;        # 包含外部配置文件
    default_type  application/octet-stream;  # 默认MIME类型
    sendfile      on;                # 启用高效文件传输
    keepalive_timeout 65;            # 长连接超时时间
}
```

### 2. Gzip 压缩配置

```nginx
gzip on;                    # 启用压缩
gzip_min_length 1k;         # 最小压缩文件大小
gzip_comp_level 2;          # 压缩级别(1-9)
gzip_types text/plain ...;  # 需要压缩的MIME类型
gzip_vary on;               # 添加Vary: Accept-Encoding头
```

### 3. Map 指令（变量映射）

```nginx
map $http_upgrade $connection_upgrade {
    default upgrade;   # 如果$http_upgrade有值且不是''，则$connection_upgrade='upgrade'
    ''      close;     # 如果$http_upgrade为空，则$connection_upgrade='close'
}
```

**语法**：`map $variable $new_variable { pattern result; }`

### 4. Server 块配置

```nginx
server {
    listen 8686 ssl;           # 监听HTTPS
    listen [::]:8686 ssl;      # 监听IPv6
    server_name example.com;    # 域名（多个用空格分隔）
    server_tokens off;          # 隐藏Nginx版本号
    
    client_max_body_size 100m;  # 最大请求体大小
}
```

### 5. SSL/TLS 配置

```nginx
ssl_certificate /path/cert.crt;           # 证书文件
ssl_certificate_key /path/key.pem;        # 私钥文件
ssl_protocols TLSv1.2 TLSv1.3;            # 启用的TLS版本
ssl_ciphers HIGH:!aNULL:!MD5;             # 加密套件
ssl_prefer_server_ciphers on;              # 优先使用服务器端套件
```

### 6. Location 块语法

```nginx
location [modifier] uri {
    # 配置指令
}
```

#### Location 修饰符：

| 修饰符 | 匹配规则 | 示例 | 优先级 |
|:---|:---|:---|:---|
| `=` | 精确匹配 | `location = /index.html` | 最高 |
| `^~` | 前缀匹配（不再检查正则） | `location ^~ /static/` | 高 |
| `~` | 区分大小写的正则 | `location ~ \.php$` | 中 |
| `~*` | 不区分大小写的正则 | `location ~* \.jpg$` | 中 |
| 无修饰符 | 前缀匹配 | `location /api` | 最低 |

### 7. Proxy Pass 语法

```nginx
location / {
    proxy_pass https://k8s-ingress;        # 代理到后端
    
    # 代理超时设置
    proxy_send_timeout 300s;               # 发送超时
    proxy_read_timeout 300s;               # 读取超时
    proxy_connect_timeout 300s;            # 连接超时
    
    # 请求头设置
    proxy_set_header Host $host;           # 传递原始Host头
    proxy_set_header X-Real-IP $remote_addr;        # 客户端真实IP
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;  # 代理链IP
    
    # WebSocket支持
    proxy_http_version 1.1;                # 必须使用HTTP/1.1
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection $connection_upgrade;
    
    # 缓冲控制
    proxy_buffering off;                   # 关闭缓冲
    proxy_cache off;                       # 关闭缓存
}
```

### 8. 变量系统

#### 常用内置变量：

| 变量 | 说明 | 示例值 |
|:---|:---|:---|
| `$remote_addr` | 客户端IP地址 | `192.168.1.100` |
| `$remote_port` | 客户端端口 | `54321` |
| `$host` | 请求中的Host头 | `example.com` |
| `$uri` | 当前请求URI（不带参数） | `/api/user` |
| `$args` | URL参数部分 | `id=1&name=test` |
| `$http_upgrade` | 请求中的Upgrade头 | `websocket` |
| `$proxy_add_x_forwarded_for` | X-Forwarded-For + remote_addr | `192.168.1.100, 10.0.0.1` |

## 五、特殊语法元素

### 1. 变量引用
```nginx
$variable_name          # 引用变量
${variable}_suffix      # 明确边界
```

### 2. 字符串
```nginx
'单引号字符串'    # 不支持转义
"双引号字符串"    # 支持转义如 \n \t
```

### 3. 时间单位
```nginx
30s   # 秒
15m   # 分钟
24h   # 小时
7d    # 天
```

### 4. 大小单位
```nginx
1k    # 1024字节
2m    # 2兆字节
3g    # 3吉字节
```

### 5. 正则表达式
```nginx
location ~ \.php$ {        # ~表示使用正则
    if ($uri ~* "^/admin") {  # ~*不区分大小写
        return 403;
    }
}
```

## 六、配置上下文层级

```nginx
main                    # 最外层
├── events {}          # 事件配置
├── stream {}          # 四层代理
│   ├── upstream {}    # 后端服务器组
│   └── server {}      # 监听服务器
└── http {}            # 七层代理
    ├── upstream {}    # 后端服务器组
    ├── server {}      # 虚拟主机
    │   ├── location {}   # URL路由
    │   └── location {}
    └── include        # 文件包含
```

## 七、常用运算符和判断

```nginx
# 变量判断
if ($request_method = POST) { ... }
if ($uri ~* \.php$) { ... }
if (-f $request_filename) { ... }  # 文件存在

# 逻辑组合（使用变量）
set $flag "0";
if ($condition1) { set $flag "1"; }
if ($condition2) { set $flag "${flag}1"; }
if ($flag = "01") { ... }
```

## 八、配置文件包含

```nginx
include /etc/nginx/conf.d/*.conf;  # 通配符包含
include mime.types;                 # 相对路径包含
```

## 语法总结

1. **分号结尾**：每个指令必须以分号`;`结束
2. **花括号**：块级配置使用`{}`
3. **空格分隔**：指令名和参数用空格分隔
4. **注释**：只支持`#`单行注释
5. **变量**：以`$`开头，支持赋值和引用
6. **上下文**：配置指令只在特定作用域生效

这些语法规则共同构成了Nginx强大而灵活的配置体系，理解它们是进行高级配置调优的基础。
