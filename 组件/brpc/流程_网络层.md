

## 网络传输层详解

网络传输层是 brpc 最复杂、最核心的部分，主要包含以下组件：

### 核心组件

| 组件 | 职责 |
|:---|:---|
| `Socket` | 连接抽象，封装 fd，管理读写缓冲区，支持 TCP/UDP/UNIX Domain Socket |
| `EventDispatcher` | 事件分发器，基于 epoll/kqueue/IOCP 实现多路复用 |
| `InputMessenger` | 处理读事件，解析协议，分发消息到对应协议解析器 |
| `SocketMap` | 管理 Socket 连接池，实现连接复用和负载均衡 |
| `IOBuf` | 零拷贝缓冲区，采用引用计数和块链式结构 |
| `WriteRequest` | 写请求队列，支持批量写和异步发送 |

## 核心代码流程详解

### 1. 读事件处理入口

```cpp
// src/brpc/socket.cpp
void Socket::StartInputEvent(SocketId id, uint32_t events) {
    // 1. 根据 id 获取 Socket 对象（引用计数+1）
    SocketUniquePtr s;
    if (GetSocket(id, &s) != 0) {
        return;
    }
    
    // 2. 调用协议特定的消息处理器
    InputMessenger* messenger = static_cast<InputMessenger*>(s->user());
    messenger->OnNewMessages(s.get());
}
```

### 2. 消息解析与分发

```cpp
// src/brpc/input_messenger.cpp
void InputMessenger::OnNewMessages(Socket* socket) {
    // 1. 循环读取，直到没有完整消息
    while (true) {
        // 2. 读取数据到 IOBuf
        ssize_t nr = socket->DoRead(&read_buf);
        
        // 3. 查找消息边界（协议相关）
        ParseResult result = policy->Parse(&read_buf, socket);
        
        if (result.is_ok()) {
            // 4. 获取完整消息，提交处理
            ProcessNewMessage(result.message(), socket);
        } else if (result.error() == PARSE_ERROR_NOT_ENOUGH_DATA) {
            // 数据不足，等待下次读事件
            break;
        }
    }
}
```

### 3. 写操作核心逻辑

```cpp
// src/brpc/socket.cpp
int Socket::Write(WriteRequest* req) {
    // 1. 将写请求加入队列（无锁队列，使用 Socket 的 _id 作为标识）
    WriteRequest* const prev_head = _write_head.exchange(req, butil::memory_order_relaxed);
    
    if (prev_head == NULL) {
        // 2. 当前无其他写操作，尝试直接写
        int saved_errno = 0;
        ssize_t nw = WriteIfConnected(req->data, &saved_errno);
        
        if (nw < 0) {
            // 3. 写失败，加入 epoll 等待可写事件
            StartWriteEvent();
        }
    }
    // 4. 有其他写操作正在处理，由它负责后续写入
    return 0;
}
```

## 关键设计亮点

| 设计 | 说明 |
|:---|:---|
| **零拷贝 IOBuf** | 数据在多层传递中不复制，通过引用计数管理生命周期 |
| **批量读写** | 多个小请求合并为一次系统调用，减少内核态切换 |
| **bthread 调度** | 用户态线程模型，避免阻塞系统线程，支持百万级并发 |
| **连接复用** | SocketMap 管理连接池，单连接多路复用 |
| **无锁队列** | 写队列使用原子操作，减少锁竞争 |
| **延迟创建连接** | 首次请求时才建立 TCP 连接，支持健康检查 |

---

## 总结

brpc 网络传输层的核心设计哲学是**高性能 + 高并发**：

1. **事件驱动**：基于 epoll/kqueue 的 Reactor 模式
2. **零拷贝**：IOBuf 贯穿整个数据流
3. **协程调度**：bthread 实现 M:N 线程模型
4. **批量优化**：读写都支持批量处理
5. **无锁设计**：关键路径使用原子操作和无锁队列

这种设计使得 brpc 在同等硬件条件下，能够达到远超传统 RPC 框架的吞吐量和延迟表现。
