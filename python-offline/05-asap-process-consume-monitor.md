# asap-process-consume-monitor 模块详解

## 概述

`asap-process-consume-monitor` 是进程资源监控服务，监控服务器进程 CPU/内存/IO 使用情况，记录到日志文件并自动清理历史日志。

## 目录结构

```
asap-process-consume-monitor/
├── monitor.py     # CPU/内存监控主逻辑
└── io_monitor.py # IO 监控
```

## monitor.py - CPU/内存监控

### 主要功能

| 函数 | 功能 |
|------|------|
| `get_cpu_usage()` | 获取 CPU 百分比和核心数 |
| `get_mem_usage_no_free()` | 获取内存使用百分比 |
| `list_process()` | 遍历进程列表，按 CPU 或内存排序 |
| `list_processes_v2()` | 使用 `ps` 命令获取进程信息 |
| `record_results()` | 格式化输出结果 |
| `write_to_file()` | 写入日志目录 |
| `clean_history_log()` | 每天凌晨 1 点清理 7 天前的日志 |

### 日志输出目录

- CPU 日志: `/data/asap/asap-process-consume-monitor/logs/cpu/`
- 内存日志: `/data/asap/asap-process-consume-monitor/logs/mem/`

### 命令行参数

```bash
python3 monitor.py [选项]
```

| 参数 | 说明 |
|------|------|
| `-n` | TopN 数量（默认 20） |
| `-m` | 指标类型: `cpu` / `mem` / `io` / `a`（全部） |
| `-cl` | 命令显示长度 |
| `-t` | IO 采样间隔（秒） |

### 日志清理

- 清理时间: 每天凌晨 1 点
- 保留期限: 7 天前的日志

## io_monitor.py - IO 监控

### 主要功能

| 函数 | 功能 |
|------|------|
| `get_process_io_rates()` | 两次采样计算 IO 速率（KB/s） |
| `get_topn_process()` | 获取读/写/总 IO 最高的 TopN 进程 |
| `get_disk_stats()` | 获取各磁盘设备读写速度和利用率 |
| `get_formatted_process()` | 格式化输出（读 Top、写 Top、IO 总 Top、磁盘状态） |

### 输出内容

1. **读 IO TopN**: 读取速率最高的进程
2. **写 IO TopN**: 写入速率最高的进程
3. **总 IO TopN**: 总 IO 最高的进程
4. **磁盘状态**: 各磁盘设备的读写速度和利用率

### IO 采样机制

通过两次采样计算 IO 速率：
```
IO速率 = (第二次采样值 - 第一次采样值) / 采样间隔
```

## 使用场景

1. **性能分析**: 识别消耗资源最多的进程
2. **容量规划**: 了解资源使用趋势
3. **故障排查**: IO 异常导致性能问题时分析来源
4. **自动化运维**: 自动清理历史日志，无需人工干预
