# python-offline 模块总览

## 概述

`python-offline` 是 ASAP/IPaaS 分布式监控系统的 Python 工具集，提供分布式集群的状态检查、资源监控、日志清理、设备管理等运维功能。

## 模块列表

| 模块 | 路径 | 功能 |
|------|------|------|
| **maxs_status_check** | `maxs_status_check/` | 核心状态检查模块，支持 Redis、MySQL、Zookeeper、Kafka、Flink、Hadoop、Spark、K8s 等 20+ 组件的检查 |
| **asap-monitor** | `asap-monitor/` | 系统指标采集和 Flink TaskManager 监控 |
| **asap-deviceid-report** | `asap-deviceid-report/` | 设备 ID 上报服务，用于软件授权验证 |
| **asap-log-purge** | `asap-log-purge/` | 日志清理服务，防止磁盘空间耗尽 |
| **asap-process-consume-monitor** | `asap-process-consume-monitor/` | 进程资源消费监控（CPU/内存/IO） |
| **sh** | `sh/` | 部署脚本目录（pre/post/scale） |

---

## 模块详解

### 1. maxs_status_check - 状态检查模块

**入口**: `maxs_status_check_tool.py`

**命令**:
```bash
python3 maxs_status_check_tool.py <maxs|rds> <base|all>
```

**检查流程**:
```
1. 基础检查 (所有服务器)
   ├── CPU 使用率检查
   ├── 内存使用率检查
   ├── 磁盘使用率检查
   └── 集群时间同步检查 (需 3 台以上)

2. 中间件检查
   ├── Redis
   ├── MySQL
   ├── DM8 / VastBase (达梦数据库)
   ├── Zookeeper
   └── Kafka

3. 计算组件检查
   ├── Flink (非 yarn 模式)
   ├── Hadoop
   ├── Spark
   └── Livy

4. Kubernetes 检查
   ├── Web UI 访问性
   ├── K8s 集群状态
   └── kube-state-metrics

5. 可观测性组件检查
   ├── ClickHouse
   ├── Prometheus
   ├── PushGateway
   ├── Minio
   ├── Elasticsearch
   └── Etcd (需 2 台以上)

6. 自定义检查
   └── 从 /data/asap/check/business_check.txt 读取额外检查项
```

**核心组件**:
- `common/hosts.py` - Ansible inventory 主机发现
- `common/call_remote.py` - SSH 远程命令执行
- `common/machine.py` - CPU/内存/磁盘检查
- `conf.ini` - 组件配置（地址、端口、凭据、命令）

---

### 2. asap-monitor - 系统监控模块

**核心脚本**:
- `perfstats-to-syslog.py` - 系统指标采集（CPU/内存/磁盘/网络/RAID）
- `flink_monitor.sh` - Flink 集群健康监控
- `checkTaskmanager.py` - TaskManager 在线检查

**数据上报**:
- 本地日志: `/data/asap/logs/asap_monitor.log`
- 数据库表: `SSA.MONITOR_MEM`, `SSA.MONITOR_DISK`, `SSA.MONITOR_IO_DISK`, `SSA.MONITOR_NETWORK`, `SSA.MONITOR_RAID`

**systemd 服务**: `asap_monitor.service`

---

### 3. asap-deviceid-report - 设备 ID 上报

**功能**: 采集服务器硬件信息生成唯一设备 ID，上报到 Redis 用于软件授权验证。

**工作流程**:
```
1. 获取设备ID (根据产品类型使用不同算法)
2. 查询数据库许可证
3. 检查设备ID重复 (防盗用)
4. 上报Redis
5. 清理过期数据
```

**配置**: `deviceid_report.cfg` (数据库、Redis、产品信息)

---

### 4. asap-log-purge - 日志清理

**功能**: 当磁盘使用率超过阈值时，自动清理各服务日志文件。

**清理策略**:
| 服务 | 方式 |
|------|------|
| spark/flink/livy/hadoop/kafka | 清空目录内所有日志 |
| elasticsearch | 清空主日志 + gc.log* |
| 其他 | 清空单个日志文件 |

**触发条件**: 磁盘使用率 > `SSA.SYS_CFG.LOG_DELETE.DISK_PERCENT`

---

### 5. asap-process-consume-monitor - 进程资源监控

**脚本**:
- `monitor.py` - CPU/内存监控
- `io_monitor.py` - IO 监控

**输出目录**:
- `/data/asap/asap-process-consume-monitor/logs/cpu/`
- `/data/asap/asap-process-consume-monitor/logs/mem/`

**命令行**:
```bash
python3 monitor.py -n 20 -m cpu   # Top20 CPU进程
python3 monitor.py -n 20 -m mem   # Top20 内存进程
python3 io_monitor.py            # IO监控
```

---

### 6. sh - 部署脚本

**目录结构**:
```
sh/
├── pre/          # 部署前准备
├── post/         # 部署后检查
└── scale/        # 扩容脚本
```

**pre.sh 功能**:
- 配置模板替换 (DB0, REDIS 等占位符)
- 组件复制到目标目录
- 动态配置修改

**post/check.sh 功能**:
- 验证部署结果
- Ansible 批量执行检查

## 通用设计模式

### 主机发现模式
```python
# 通过 ansible-inventory 获取主机分组
cmd = 'ansible-inventory -i /data/maxs-ops/inventory/hosts --list'
host_json = json.loads(subprocess.Popen(...))
```

### 远程执行模式
```python
# 通过 call_remote.py 在远程主机执行命令
result = call_remote.call_remote_server(host, check_cmd)
```

### 检查结果格式
```
**[id] Check Component........PASS**
**[id] Check Component........FAIL**
```

### 错误收集机制
```python
error = []  # 收集所有失败的检查项
if len(error) > 0:
    log.error('Check failed. The processes with problems include:{}'.format(str(error)))
```
