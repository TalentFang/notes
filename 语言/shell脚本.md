# 常见语法

## 一、特殊变量：`$?` 和 `$@`

### `$?` —— 上条命令的退出状态码
- **含义**：返回上一个执行命令的退出状态（0 表示成功，非0 表示失败）
- **用途**：判断命令是否执行成功

```bash
#!/bin/bash

ls /home
echo "上条命令状态: $?"   # 输出 0（成功）

ls /不存在的目录
echo "上条命令状态: $?"   # 输出 2（失败）

# 实际应用
if [ $? -eq 0 ]; then
    echo "执行成功"
else
    echo "执行失败"
fi
```

### `$@` —— 所有位置参数（推荐）
- **含义**：表示传递给脚本的所有参数（`$1`, `$2`, `$3`...）
- **特点**：每个参数作为独立的字符串，保留原始边界
- **对比**：`$*` 将所有参数合并为一个字符串

```bash
#!/bin/bash
# 假设脚本名为 test.sh，执行: ./test.sh "hello world" foo bar

echo "参数总数: $#"      # 输出 3

echo '使用 $@:'
for arg in "$@"; do
    echo "[$arg]"
done
# 输出:
# [hello world]
# [foo]
# [bar]

echo '使用 $*:'
for arg in "$*"; do
    echo "[$arg]"
done
# 输出:
# [hello world foo bar]   # 合并成一个字符串
```

**总结**：**几乎总是用 `"$@"`**，它能正确处理带空格的参数。

## 二、引号的使用规则

### 三种引号的区别

| 引号 | 名称 | 特性 | 示例 |
|------|------|------|------|
| `" "` | 双引号 | **弱引用**：允许变量扩展 `$var` 和命令替换 `$(cmd)` | `"Hello $USER"` |
| `' '` | 单引号 | **强引用**：原样输出，不解析任何特殊字符 | `'Hello $USER'` |
| `` ` `` | 反引号 | 命令替换（旧语法，建议用 `$()`） | `` `date` `` |
> **"变量扩展和命令替换，永远用双引号包裹，除非你知道自己在做什么。"

## 三、其他重要特殊变量

| 变量 | 含义 | 示例 |
|------|------|------|
| `$0` | 脚本名称 | `./test.sh` |
| `$1~$9` | 第1-9个参数 | `$1` 是第一个参数 |
| `${10}` | 第10个及以上参数 | 必须用花括号 |
| `$#` | 参数个数 | `if [ $# -lt 2 ]` |
| `$$` | 当前进程PID | 创建临时文件：`/tmp/tmp.$$` |
| `$!` | 最后后台进程PID | `sleep 10 & echo $!` |
| `$_` | 上条命令的最后一个参数 | 快速复用 |
