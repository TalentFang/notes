# PostgreSQL 核心概念

## 数据库架构

```
PostgreSQL 集群 (PGDATA)
├── 数据库 (Database)
│   ├── Schema (模式)
│   │   ├── 表 (Table)
│   │   ├── 索引 (Index)
│   │   ├── 视图 (View)
│   │   ├── 函数 (Function)
│   │   └── 序列 (Sequence)
│   └── 其他对象
└── 角色/用户
```

## 核心概念

### 集群 (Cluster)
PostgreSQL 实例管理的所有数据库的集合。

### 数据库 (Database)
独立的数据库环境，每个数据库有自己的 schema 集合。

### Schema（模式）
命名空间，用于组织表、视图、函数等对象。
- 类似于 MySQL 的 Database
- 默认 schema：`public`

### 表 (Table)
数据存储的基本单位。
```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100),
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 索引 (Index)
加速查询的数据结构。
```sql
CREATE INDEX idx_name ON users(name);
```

### 角色 (Role) 和 用户 (User)
- Role 可以登录（LOGIN）
- User 是带 LOGIN 属性的 Role
- 权限系统基于角色继承

## PostgreSQL vs MySQL 对比

| 概念 | PostgreSQL | MySQL |
|---|---|---|
| 数据库 | 集群 | 实例 |
| Schema | 命名空间 | 相当于 Database |
| 大小写 | 表名区分大小写（默认） | 表名不区分（Windows） |
| 默认端口 | 5432 | 3306 |
| 存储引擎 | 单一引擎 | 多种引擎（InnoDB等） |

## 关键特性

### 1. MVCC（多版本并发控制）
- 读写不阻塞
- 通过版本号实现事务隔离

### 2. WAL（预写日志）
- 数据安全性高
- 支持 point-in-time 恢复

### 3. MVCC + SSI
支持 SERIALIZABLE 隔离级别（最高）

### 4. 扩展性
- 支持自定义类型
- 支持自定义函数
- 丰富的数据类型（JSON、ARRAY、GIS）

## 进程结构

```
PostgreSQL 进程
├── postgres (主进程)
│   ├── backend (客户端连接)
│   │   └── postgres 连接处理
│   ├── wal writer (写日志)
│   ├── autovacuum (垃圾回收)
│   └── stats collector (统计信息)
```
