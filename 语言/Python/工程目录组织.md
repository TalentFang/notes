# 工程目录组织

## 常见目录结构

### 方式1: 简单项目

```
my_project/
├── my_package/
│   ├── __init__.py
│   ├── module1.py
│   └── module2.py
├── tests/
│   ├── __init__.py
│   ├── test_module1.py
│   └── test_module2.py
├── docs/
├── requirements.txt
├── README.md
└── setup.py  # 或 pyproject.toml
```

### 方式2: 复杂项目 (推荐)

```
my_project/
├── src/                    # 源代码目录 (避免项目根目录污染)
│   └── my_package/
│       ├── __init__.py
│       ├── core/
│       │   ├── __init__.py
│       │   └── logic.py
│       ├── api/
│       │   ├── __init__.py
│       │   └── routes.py
│       └── utils/
│           ├── __init__.py
│           └── helpers.py
│
├── tests/                  # 测试
│   ├── unit/
│   ├── integration/
│   └── fixtures/
│
├── docs/                   # 文档
├── scripts/               # 工具脚本
├── configs/               # 配置文件
│
├── .gitignore
├── requirements.txt       # 依赖列表
├── requirements-dev.txt   # 开发依赖
├── requirements-test.txt  # 测试依赖
├── pyproject.toml         # 现代 Python 项目配置
├── setup.cfg              # 旧版配置
└── README.md
```

### 方式3: Flask/Django 项目

```
flask_app/
├── app/
│   ├── __init__.py        # Flask 应用工厂
│   ├── models.py
│   ├── views/
│   │   ├── __init__.py
│   │   └── main.py
│   ├── templates/         # HTML 模板
│   └── static/           # CSS, JS, 图片
├── migrations/            # 数据库迁移
├── tests/
├── requirements.txt
├── config.py             # 配置
└── run.py                # 启动入口
```

### 方式4: ASAP 项目结构

```
python-offline/            # 项目根目录
├── maxs_status_check/    # 功能模块
│   ├── __init__.py
│   ├── maxs_status_check_tool.py  # 入口点
│   ├── common/           # 公共模块
│   │   ├── __init__.py
│   │   ├── hosts.py
│   │   ├── machine.py
│   │   ├── redis.py
│   │   └── ...
│   └── conf.ini          # 配置文件
├── asap-monitor/         # 另一个功能模块
├── asap-deviceid-report/
├── tests/                # 共享测试 (如果存在)
└── setup.py
```

## 关键文件

### `__init__.py` - 模块标识

每个包目录需要有 `__init__.py`，使该目录成为 Python 包。

```python
# 最小化
# 只需存在，不需内容

# 完整版本 - 统一导出
from .module1 import PublicClass, public_function
from .module2 import AnotherClass

__all__ = ['PublicClass', 'public_function', 'AnotherClass']
__version__ = "1.0.0"
```

### `pyproject.toml` - 现代项目配置

```toml
[build-system]
requires = ["setuptools>=45", "wheel", "setuptools_scm[toml]>=6.2"]
build-backend = "setuptools.build_meta"

[project]
name = "my-package"
version = "1.0.0"
description = "A sample package"
authors = [{name = "Example", email = "example@example.com"}]
dependencies = ["requests>=2.28"]

[project.optional-dependencies]
dev = ["pytest", "black", "mypy"]

[tool.black]
line-length = 88
```

### `setup.py` / `setup.cfg` - 打包配置

```python
# setup.py
from setuptools import setup, find_packages

setup(
    name="my-package",
    version="1.0.0",
    packages=find_packages(where="src"),
    package_dir={"": "src"},
    install_requires=["requests>=2.28"],
    python_requires=">=3.8",
)
```

## 与 Java 对比

| Python | Java |
|--------|------|
| `src/` (可选) | `src/main/java/` (固定) |
| `__init__.py` | `package` 声明 |
| `pyproject.toml` | `pom.xml` / `build.gradle` |
| `requirements.txt` | `pom.xml` dependencies |
| 入口脚本: `__main__.py` | 入口类: `public static void main` |
